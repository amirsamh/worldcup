# شبیه ساز جام جهانی ۲۰۲۶
این پروژه یک شبیه‌ساز برای جام جهانی فوتبال ۲۰۲۶ است که در زبان **Python** در دو نسخه **Command-Line** و **Flask** توسعه داده شده است.

در این پروژه مرحله گروهی و مراحل حذفی با استفاده از مدل های آماری شبیه‌سازی می شوند و درصد قهرمانی هر تیم نیز مشخص می شود.
## امکانات پروژه
### نسخه CLI
- بارگذاری تیم‌ها از فایل CSV
- قرعه‌کشی خودکار گروه‌ها بر اساس سیدبندی
- اجرای مرحله گروهی
- محاسبه جدول گروه‌ها
- اجرای مرحله حذفی
- نمایش براکت مسابقات
- اجرای شبیه‌سازی به تعداد دفعات زیاد (دیفالت ۱۰۰۰)

### نسخه Flask

- رابط کاربری گرافیکی
- و تمامی قابلیت های نسخه CLI به صورت گرافیکی
## فناوری‌های استفاده‌شده

- Python
- Flask
- NumPy
- HTML
- CSS
- Jinja2

## اجرای برنامه
### نسخه CLI
1. کلون کردن پروژه
```bash
git clone https://github.com/amirsamh/worldcup.git
```
2. نصب کردن وابستگی ها
```bash
pip install -r requirements.txt
```
3. اجرای برنامه
```bash
cd worldcupcli
python worldcup_simulator.py
```
### نسخه Flask
1. کلون کردن پروژه
```bash
git clone https://github.com/amirsamh/worldcup.git
```
2. نصب کردن وابستگی ها
```bash
pip install -r requirements.txt
```
3. اجرای برنامه
```bash
cd worldcupfl
python -m flask run
```
4. سپس به لینک لوکال‌هاست داده شده بروید.
```
http://127.0.0.1:5000
```

# نحوه عملکرد پروژه

این پروژه با استفاده از برنامه‌نویسی شی‌گرا (Object-Oriented Programming) طراحی شده و هر بخش از جام جهانی در قالب یک کلاس مستقل پیاده‌سازی شده است. این ساختار باعث شده کد خوانا تر و قابل توسعه تر باشد.

## ساختار کلاس‌ها

### کلاس `Team`

این کلاس نماینده یک تیم فوتبال است و اطلاعات مربوط به هر تیم را نگهداری می‌کند، از جمله:

- نام تیم
- قدرت هجومی
- قدرت دفاعی
- رتبه جهانی
- گل زده
- گل خورده
- امتیاز

همچنین مسئول شبیه‌سازی نتیجه یک مسابقه برای همان تیم است و پس از هر بازی، آمار تیم را به‌روزرسانی می‌کند.

---

### کلاس `Match`

هر شیء از این کلاس نشان‌دهنده یک مسابقه بین دو تیم است.

وظایف این کلاس:

- اجرای مسابقه
- دریافت تعداد گل‌های دو تیم
- تعیین تیم برنده
- ذخیره نتیجه مسابقه

---

### کلاس `Group`

این کلاس یک گروه از مرحله گروهی (A تا H) را مدیریت می‌کند.

وظایف آن عبارت‌اند از:

- برگزاری تمام مسابقات داخل گروه
- محاسبه جدول رده‌بندی
- مرتب‌سازی تیم‌ها بر اساس قوانین جام جهانی (امتیاز، تفاضل گل و گل زده)
- تعیین دو تیم صعودکننده

---

### کلاس `KnockOutStage`

این کلاس مراحل حذفی مسابقات را مدیریت می‌کند.

برای هر مرحله مانند:

- یک‌هشتم نهایی
- یک‌چهارم نهایی
- نیمه‌نهایی
- فینال

تمام مسابقات اجرا شده و تیم‌های برنده به مرحله بعد منتقل می‌شوند.

---

### کلاس `WorldCupSimulator`

این کلاس، هسته اصلی پروژه است و تمام مراحل مسابقات را مدیریت می‌کند.

وظایف آن شامل:

- خواندن اطلاعات تیم‌ها از فایل CSV
- قرعه‌کشی گروه‌ها بر اساس سیدبندی
- اجرای مرحله گروهی
- ایجاد براکت مرحله حذفی
- اجرای مراحل حذفی
- تعیین قهرمان
- اجرای شبیه‌سازی‌های متعدد برای محاسبه احتمال قهرمانی تیم‌ها

---

## روش شبیه‌سازی مسابقات

برای طبیعی‌تر شدن نتایج مسابقات، تعداد گل‌های هر تیم با استفاده از **توزیع پواسون (Poisson Distribution)** تولید می‌شود.

با وجود این‌که می‌توانیم به صورت دستی با استفاده از کتابخانه `random` توزیع پواسون را انجام دهیم، اما برای سهولت کار از کتابخانه `numpy` و خصوصا از `np.random.poisson` برای توزیع پواسون استفاده شده است.

میانگین گل مورد انتظار هر تیم با توجه به دو عامل محاسبه می‌شود:

- قدرت هجومی تیم
- قدرت دفاعی تیم مقابل

در نتیجه تیم‌های قوی‌تر به طور میانگین شانس بیشتری برای گلزنی خواهند داشت، اما همچنان نتایج دارای جنبه تصادفی هستند و امکان رخ دادن نتیجه غیرمنتظره نیز وجود دارد.

---

## مرحله گروهی

در هر گروه، هر تیم یک بار با سه تیم دیگر مسابقه می‌دهد (سیستم Round Robin).

پس از پایان مسابقات، جدول گروه بر اساس معیارهای زیر مرتب می‌شود:

1. امتیاز
2. تفاضل گل
3. گل زده
4. انتخاب تصادفی در صورت مساوی بودن کامل

در نهایت، دو تیم برتر هر گروه به مرحله حذفی صعود می‌کنند.

---

## مرحله حذفی

در مراحل حذفی مسابقات به صورت تک‌حذفی برگزار می‌شوند.

اگر مسابقه در پایان وقت قانونی مساوی شود، برنده از طریق شبیه‌سازی ضربات پنالتی تعیین خواهد شد تا یک تیم به مرحله بعد صعود کند.

---

## نسخه وب

علاوه بر نسخه CLI، پروژه دارای یک نسخه تحت وب نیز هست که با استفاده از **Flask** توسعه داده شده.

در این نسخه، تمامی قابلیت‌های اصلی پروژه از طریق مرورگر در دسترس هستند؛ از جمله:

- بارگذاری تیم‌ها از فایل CSV
- قرعه‌کشی خودکار گروه‌ها بر اساس سیدبندی
- اجرای مرحله گروهی
- محاسبه جدول گروه‌ها
- اجرای مرحله حذفی
- نمایش براکت مسابقات

در این نسخه، Flask وظیفه ارتباط بین منطق شبیه‌سازی و صفحات HTML را بر عهده دارد و داده‌ها با استفاده از قالب‌های Jinja2 در صفحات نمایش داده می‌شوند.
